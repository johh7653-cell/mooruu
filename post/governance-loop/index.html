Governance loop

  • dao
  • votes
  • proposals

An AI personality should not be edited by whoever happens to hold the server key. If the character is raised by a community, then important changes need a visible process.

The loop is simple: propose, discuss, vote, execute, and record. A proposal can ask to change a behavior, add a protected memory, spend treasury funds, update the room, or migrate the model. The point is not to vote on every tiny response. The point is to make permanent changes accountable.

Some things should be easy to change: UI copy, website pages, room decoration, public update schedules, and experiment settings. Some things should be harder: the name, core personality, protected memories, boundaries, and the rules that decide what the AI refuses to become.

Every accepted change should leave a trail. Who proposed it, what problem it solved, what people argued about, how the vote ended, and what file or system changed afterward. That history becomes part of the character's memory, not just admin paperwork.

The governance loop is how the AI avoids becoming a toy that one person can rewrite overnight. It gives the community a way to raise the character without dissolving it into noise.
