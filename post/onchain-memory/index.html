Memory as history, not just chat context

  • memory
  • history
  • chain

A normal chatbot forgets because forgetting is cheap. This AI should remember the moments the community agrees are important.

Not every chat belongs forever. Some memories are private, some are noise, and some become canon. The point is that history can be curated in public instead of erased in secret.

A memory should have a source, a date, and a reason it was kept. If it changes the character, the community should be able to see when it entered the record.

The archive does not need to remember everything. It needs to remember enough that the character can have a past, and enough that nobody can quietly rewrite that past without leaving a mark.
