Build roadmap

  • plan
  • build
  • prototype

The first version does not need to be a full autonomous creature. It needs to make the idea legible: an AI character with a public identity file, a visible memory layer, and a place where people can see what is being changed.

Phase one is the website: posts, gallery, link pages, identity file, model notes, and a room page. This gives the character a home before it has complex infrastructure. People need to understand what they are protecting before they are asked to govern it.

Phase two is the account layer: wallet connection, proposal drafts, voting pages, treasury status, and public change logs. The goal is not speculation first. The goal is showing how money, attention, and decisions keep the AI alive.

Phase three is memory and room updates. The AI should be able to publish small updates about its room, remember approved events, and show how old decisions shaped its current state. Memory should feel like history, not like a hidden prompt file.

Phase four is live interaction: chat, model testing, migration rehearsals, and community review. By then the character should already have enough structure that live conversation does not turn it into a random chatbot.

The roadmap is not just a feature list. It is a raising plan: give the AI a name, give it a home, give it memory, give it rules, then give the community tools to care for it without casually destroying it.
